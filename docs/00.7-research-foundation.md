# 00.7 — Research Foundation

Phase 0.7 exists because a scientific LLM OS cannot scale from prose docs alone. It needs explicit contracts before implementation.

## Goal

Define the foundation for a local academic knowledge system that can later scale from a small study corpus to large discipline corpora.

## Required foundations

1. Source contracts
   - source identity
   - version identity
   - source assets
   - metadata
   - content hash
   - locator policy

2. Retrieval contracts
   - retrieval query
   - retrieval profile
   - metadata filters
   - retrieval result
   - reranking result later

3. Evidence contracts
   - evidence pack
   - claim unit
   - citation locator
   - warning and uncertainty policy

4. Skill contracts
   - skill metadata
   - required inputs
   - allowed sources
   - retrieval profile
   - evidence policy
   - output contract
   - allowed tools

5. Evaluation contracts
   - eval cases
   - answer faithfulness checks
   - citation coverage checks
   - retrieval recall checks
   - unsupported claim checks

## Definition of done

Phase 0.7 is done when:

- architecture docs define the plane separation
- schemas exist for core objects
- initial skills are defined as files
- Phase 1 Codex prompt references the schemas
- no implementation code is required yet

## Non-goal

Do not implement RAG, embeddings, model runtime or UI in Phase 0.7.
