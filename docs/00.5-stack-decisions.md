# 00.5 — Stack Decisions Freeze

Phase 0.5 removes stack ambiguity before implementation starts.

## Locked decisions

| Area | Decision |
|---|---|
| Desktop | Tauri v2 |
| Frontend | Solid 1.x + Vite SPA |
| Backend authority | Rust Core |
| SQLite | `rusqlite` |
| Migrations | `rusqlite_migration` |
| Model runtime | llama.cpp HTTP server process later |
| Runtime ownership | Rust Runtime Supervisor later |
| Docker | Optional for CI/test/isolated services only |
| Product dependency on Pi/OMP | None |
| Pi/OMP role | Optional future external adapter/orchestrator only |
| Preferred boundary order | aegis-core / aegis-cli before any OMP adapter |
| Product dependency on MCP | None |

## Plane separation

AEGIS uses explicit architecture planes:

```text
Control Plane   = source registry, metadata, config, audit, permissions
Retrieval Plane = lexical index, vector index, reranking, metadata filtering
Model Plane     = local model runtime, prompts, synthesis, embeddings later
Skill Plane     = declarative academic task contracts
```

No plane may bypass Corpus Authority.

## Authority rule

Rust owns sensitive local authority. The frontend requests actions through Tauri commands and does not own direct execution, filesystem mutation, runtime supervision or source registry state.

## Persistence

SQLite stores local control-plane state:

- source registry indexes
- source metadata indexes
- ingestion state
- chunk metadata indexes
- evidence pack indexes
- audit events
- migration metadata

Large artifacts remain files:

- PDFs
- slides
- extracted text
- snapshots
- chunks
- evidence packs

## Exclusions

AEGIS v1 core excludes:

```text
Pi
MCP
Aider
OpenCode dependency
external coding harness dependency
```

These may be used by the developer externally, but AEGIS must not require them to run product features.
